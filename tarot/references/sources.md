# Sources And Reference Notes

This skill is inspired by Rider-Waite-Smith tarot conventions and reflective tarot consultation practice.

Confirmed references:

- `ekelen/tarot-api`: provides 78-card JSON/API data using A. E. Waite's public-domain _The Pictorial Key to the Tarot_ meanings, including upright and reversed meanings.
- `lawreka/ascii-tarot`: useful CLI reference for simple draws, three-card readings, Celtic Cross, and reversals.
- `searge/tarot`: useful Rider-Waite reference library for card metadata and image-oriented workflows.
- `MarketingPipeline/Tarot.js`: useful design reference for separating deck management, spread definitions, and reading logic.
- OpenAI Usage Policies: use as the high-level boundary for avoiding unlicensed professional advice and high-impact decision guidance.

When exact current API behavior, repository license terms, pricing, privacy, or policy details matter, check the official repository or product page before answering.

Local imported files:

- `references/source-data/ekelen-tarot-api-card_data.json`: unmodified local copy of `static/card_data.json` from `ekelen/tarot-api`.
- `references/cards/*.md`: generated per-card Markdown files containing source name, upright meaning, reversed meaning, and pictorial description.
